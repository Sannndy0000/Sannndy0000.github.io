Research

ADAPT(Advanced Data and Programming Technology) Lab, SJTU


I started a project on recognizing speakers' relationship from conversations earlier this semester and joined ADAPT for the guidance of Prof. Kenny Zhu.

A few progress has been made since then.

  • Dataset: Movie Dialogs with relationship labels
    Since large-scale transcripts of human conversations are very hard to obtain, we use dialogs from movies. We crawled 1000+ movie scripts from various websites, and left those in relatively normative format. For each script, we looked for corresponding IMDB page to merge different versions of one movie.
    one-to-one conversations are extracted from the scripts after scene splitting based on simple rules, and listed name pairs that have at least 4 conversation sessions and 20 lines. We asked human volunteers to label the relationship of those pairs after they watch the movie or read the synopsis. Relationships that involve too many changes or are vague are discarded. Now we have more than 5000 sessions with labels, across 17 classes.
  • Current Results
    • Address Terms Extraction
      • Address terms are classified into 3 categories: name, titles and placeholders (Vinodh Krishnan and Jacob Eisenstein, 2014).I designed an algorithm to extract address terms (tell address terms from referring to other names) and labeled them with the exact category, which has achieved an accuracy of 92.1% across 1000 sentences, compared with human annotations.
    • Wording and Emotion: with Linguistic Inquiry and Word Count(LIWC) features and multilayer perceptron as the classifier, I got an accuracy of 68%(18% gain over the chance accuracy)on binary(family-workplace) classification problem. Other features like address terms, Dictionary of Affect in Language(DAL), turn lengths, co-verbs(actions carried out together by the speakers) will be added to the model soon.
    • Hierarchical Attention LSTM achieved an accuracy of 73% on this task.
    I'm actively working on this topic currently. Any suggestion or idea will be greatly appreciated!

BCMI (Brain-like Computing and Machine Intelligence) Lab, SJTU


The lab aims to understand the mechanism of intelligent information processing and cognitive process in the brain and to develop new type computing structures and algorithms for information technology. My work was involved in 2 tasks:

  • Emotion Classification Using a series of movie clips as stimulator, we conducted experiments on subjects to arouse their emotion(happy or sad) while they watch the clips, and record their EEG(electroencephalogram) signals during the whole process. We built personalized EEG-based affective models with those data. I ran models with SVM(tried multiple features, and added some preprocessing steps like period-interception, noise reduction by superposing signals), LSTM, CNN and compared their results.
  • Depression Detection We want to apply our methods to help psychiatrists diagnose or better understand depression. During my summer intern, we worked on experiment design. I proposed to add cognitive tasks to original emotional tests, and used VR materials to enhance the arousal.

PintOS


PintOS is a simple operating system framework, and we strengthened its support in kernel threads, loading and running user programs, and file system.

We worked on the following tasks together(pair-programming):

  • Threads Synchronization
    implemented priority scheduling and priority donation to avoid priority inversion;
  • Loading and running user programs
    • Argument passing. Set up the stack.
    • User memory access. Validate pointers used by user programs, and terminate offending process.
    • System call infrastructure, and all the system calls including exit() and wait().
  • Virtual Memory
    We designed our implementation from scratch: supplementary page table, frame table,swap table, and their interactions to deal with memory access, page fault handling, program lazy loading, eviction policy, swap slots track and so on.
  • File System
    • An indexed on-disk inode structure with direct, indirect, and doubly indirect blocks. Implemented file growth with "sparse file" support.
    • Subdirectories, current working directory, and relevant system calls like readdir().
    • File system synchronization
September-December 2017

ACORN, a search engine with both web text and image searching(2016)


I built such a search engine from scratch(except for the front-end, which is done by my partner), including following parts:


  • Web scrawler. I crawled websites and pictures parsed with Beautifulsoup and later lxml to speed up.
  • For text-to-text and text-to-picture searching, I built dictionary and index with Lucene, and used Hadoop to improve processing efficiency. Special pre-processing like word segmentation are required for Chinese websites.
  • For image search(which means the input is a picture), I first tried out several classical algorithms to count picture similarity: Implemented Canny for edge detection. To solve the problem that the detected edges are not smooth, I designed a "track finding" algorithm to connect edge points at the gaps. Also I implemented SIFT(Scale-Invariant Feature Transform) to detect the corresponding points after a picture is amplified/shrunk/rotated.
    RGB distribution are also used as a measure of similarity. The 2 parts are combined to produce a final ranking.
